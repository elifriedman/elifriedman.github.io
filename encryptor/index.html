Expand for how the encryption works ➕

This tool uses modern authenticated encryption (AES‑256‑GCM + Argon2id) to protect your data:

  • Key Derivation: The password is turned into an encryption key using Argon2id (memory‑hard, GPU‑resistant). Fallback if Argon2id is not available: PBKDF2 (200k iterations). A warning is shown when the fallback is active.
  • Encryption: AES‑256‑GCM, which provides both confidentiality and integrity (tamper detection).
  • Randomness: Each file uses a fresh 128‑bit salt and 96‑bit nonce (counter portion zeroed before use).
  • Chunk integrity: The total chunk count is committed in the authenticated header, preventing truncation attacks.
  • Authenticated Metadata: File header (version, KDF params, salt, nonce, chunk count) is included as authenticated data.
  • File Format: MAGIC | header length | JSON header | ciphertext chunks

Security depends on a strong password and a trusted browser environment.
